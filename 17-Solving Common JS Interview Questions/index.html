1. What is the difference between undefined and not defined in JavaScript?

undefined means a variable has been declared but has not yet been assigned a value. On the other hand, not defined means that the variable itself has not been declared 

          
var x;
console.log(x); // output: undefined
console.log(y); // output: ReferenceError: y is not defined

2. What is the difference between == and ===?

== checks for equality of value without paying attention to type. === checks for equality of value AND type. 

                    
console.log(1 == '1'); //true
console.log(1 === '1'); //false

The id of this paragraph is "myParagraph"


You can add "mouseover" event to change color css property to red. you can also add another "mouseleave" event to turn the text back to black when you leave the element

7. What is the short way to create a function that takes two parameters param1 and param2 and returns the object {param1, param2}?

Normal way

function createObject(param1, param2) {
  return {
    param1: param1,
    param2: param2
  };
}

ES6 short

const createObject = (param1, param2) => ({param1, param2});

()=>{} - This is called an arrow function. It is a shorthand way of writing a function. If there is only one parameter in the function, you can omit the parentheses.

no return - This is called implicit return. It allows you to omit the return keyword when returning a single expression. If you omit the parentheses around the object, you will get an error.

{param1, param2} - This is called object literal shorthand. It allows you to omit the property value if key matches variable name.

8. Add List Item to html list

ID of this list is "myList"

  • Item 1

Option 1

const myList = document.querySelector('#myList');
const li = document.createElement('li');
li.textContent = 'Item 2';
myList.appendChild(li);

Option 2

const myList = document.querySelector('#myList');
myList.innerHTML += '<li>Item 2</li>';

Both options will add a new list item to the list. The first option is more efficient because it only adds one element to the DOM. So we can easily add things like event listeners to it. The second option adds the entire list again.

10. What is the difference between the following two functions 

          
function myFunction() {
  console.log('hello');
}

const myFunction = function() {
  console.log('hello');
};

The first one is a function declaration. The second one is a function expression. The main difference is that function declarations are hoisted and function expressions are not. 

          
functionDec(); //works

function functionDec() {
  console.log('hello');
}

functionExp(); //error

const functionExp = () => {
  console.log('hello');
};
