1. What is the output of the following code?

        
console.log(0.1 + 0.2 == 0.3);

The output of the above code will be false.
In JavaScript, 0.1 + 0.2 is equal to 0.30000000000000004 and not 0.3. This is because of the way floating point numbers are stored in JavaScript. To avoid this, it is recommended to round the numbers to a specific decimal places, before performing the arithmetic operation.
For example: 

		  
let result = 0.1 + 0.2;
result = result.toFixed(1);
console.log(result == 0.3);

let x = 5;
console.log(x++);
console.log(++x);
The output of the above code will be: 
        
5
7
When x++ is used, the value of x is used and then incremented. However, when ++x is used, the value of x is first incremented and then used.

3. What is the output of the following code?

        
let x = 10;
let y = (x > 5) ? "Hello" : "Goodbye";
console.log(y);

The output of the above code will be Hello.

The above code is using the JavaScript ternary operator. It is a short way to write an if...else statement. The syntax for the ternary operator is condition ? expr1 : expr2. If condition is true, the operator returns the value of expr1; otherwise, it returns the value of expr2.

4. What is the output of the following code?

        
console.log(3 + 4 + "5");

The output of the above code will be 75.

The above code is using the + operator for arithmetic addition and string concatenation. When the + operator is used to add two numbers, it performs arithmetic addition. However, when the + operator is used to add a number and a string, it performs string concatenation.

const multiply = x => x * x;
console.log(multiply(5));

The output of the above code will be 25.

The above code is using an arrow function, which is a new feature introduced in ES6.

since it only has one parameter, the parentheses around the parameter can be omitted.

Since the function has only one statement, and the statement returns a value, there is no need for the statement to have a return statement. or curly braces.

6. What is the output of the following code?

		
let a = [1, 2, 3];
let b = [1, 2, 3];
console.log(a === b);

The output of the above code will be false.

The above code is comparing two arrays. In JavaScript, arrays are reference types and not value types. This means that when we compare two arrays using the === operator, it will only return true if both arrays are references of the same object.

if we want to compare the values of two arrays, we need to loop through each element and compare them one by one.

7.Implement a function called uniqueArray that takes an array as input and returns a new array with only the unique elements, removing any duplicates.

example: 
		
console.log(uniqueArray([1, 2, 2, 3, 4, 4, 5])); // [1, 2, 3, 4, 5]
		
			
		
function uniqueArray(arr) {
	return arr.filter((item, index) => arr.indexOf(item) === index);
}

9. Write a function that can produce the following output on the console. The number of lines should be customizable. 

		
++++++++
+++++++-
++++++--
+++++---
++++----
+++-----
++------
+-------
--------
	
	  
		
function printPattern(n) {
	for (let i = n; i >= 0; i--) {
		let out = '+'.repeat(i);
		out = out.padEnd(n, '-');
		console.log(out);
	}
}

10. Write a function in JavaScript that checks if a given string is a palindrome.

A palindrome is a word, phrase, number, or other sequence of characters that reads the same forward and backward.

The function should ignore spaces, punctuation, and capitalization when determining if the input string is a palindrome.

It should work lilke this: 

		
isPalindrome('A man a plan a canal Panama'); // true
isPalindrome('racecar'); // true
isPalindrome('hello'); // false
		
		
		
function isPalindrome(str) {
	str = str.toLowerCase().replace(/[^a-z0-9]/g, '');
	return str === str.split('').reverse().join('');
}
